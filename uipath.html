Weather Robot

The weather robot will allow a user to enter a location (ex. city name), the robot will return the entered locations weather, it will also analyze the weather, and it will display a personalized message to the user. The activites were placed inside a Sequence container. There are four main steps in the sequence and some of the main sequences can be broken down into smaller sub-steps.

Technologies used

  • UiPath Studio
  • EI Extension

Processes

Steps

Step 1.

The first step in the process is the Input Dialog. The dialog will display its title that describes the purpose of the automation and has a label that lets the user know what to type in.

In the dialog properties field I created a Variable to store the user input. I assume the input will be a string and I expanded its scope to be accessible to all activities in the container.

Step 2.

The Sequence : Get weather opens Internet Explorer. There are five more activities within this container that can be broken down to sub-steps.

2.1 Goes to https://google.com

2.1.1 The Do sequence contains 5 more steps.

2.1.1.1 Once the browser is open, the robot will access the variable city that stored the users input and concatenate it to get weather from that city.city+"weather"

2.1.1.2 Robot clicks Enter

2.1.1.3 For the data scraping I used the Get Full activity. It will target weather condition and store that data in string variable called getCondition

2.1.1.4 This Get Full activity will target the weathers temperature and assign it to variable getTemperature

2.1.1.5 Once data scraping is complete the robot will close the browser.

Step 3.

The robot will be able to make a decision based on the user inputs feedback. I used a Flowchart that will analyze the weather and make a suggestion based on the getTemperature and getCondition variables.

3.1 The if statement will scan the getCondition variable and if it contains specific keywords like Rain, Showers, or Thunderstorm, a new variable rainGear will display a message to let the user know to bring an umbrella, else a variable notRaining will display donâ€™t bring an umbrella.

3.2 A flow decision will use getTemperature to determine if its cold getTemperature < 40 . However, this flowchart is a boolean and the getTemperature variable is a string. To convert a string to int I used the VB .toint getTemperature.Toint < 40

3.3 The second flow decision will do the same but it will determine if its too hot

Step 4.

The last steps displays the message to the user. It will display the city's weather condition, temperature and display a custom message based on the temperature and weather condition
